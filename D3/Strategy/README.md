# 策略模式

## 目的

將各種可以互換的演算法(策略), 包裝成一個類

<img src='./01.png' width='75%'>

<img src='./02.png' width='75%'>

## 適用場景

- 需要在運行時切換不同的演算法

- 當有許多僅在執行某些行為時的相似類時

## 優點

- 可以使用組合來代替繼承

- 可以將算法與使用隔離開來

## 缺點

- 客戶端必須知道算法具體不同
